## 📖 Примеры

### Пример 1: CRUD операции с FastAPI и SQLAlchemy

**Цель:** Познакомить студентов с базовыми CRUD операциями с использованием SQLAlchemy ORM.

**Что включено:**
- Модель `User` с полями: id, username, email, full_name
- FastAPI приложение с эндпоинтами для всех CRUD операций
- HTTP-файл для тестирования API в PyCharm
- Примеры валидации с Pydantic

**База данных:** `demo_crud_db`

**Ключевые концепции:**
- Определение моделей SQLAlchemy
- Использование Session для работы с БД
- Разница между моделями SQLAlchemy и Pydantic

### Пример 2: Начало работы с Alembic

**Цель:** Показать процесс инициализации и настройки Alembic.

**Что включено:**
- Настроенная структура Alembic (alembic.ini, env.py)
- Модель `Product`
- Документация по конфигурации

**База данных:** `demo_alembic_db`

**Ключевые концепции:**
- Инициализация Alembic (`alembic init`)
- Настройка подключения к БД
- Структура файлов Alembic

### Пример 3: Создание миграций

**Цель:** Научить создавать миграции автоматически и вручную.

**Что включено:**
- Модель `Article` с несколькими полями
- Готовая структура для практики
- Примеры автогенерации и ручного создания миграций

**База данных:** `demo_migrations_db`

**Ключевые концепции:**
- Автоматическая генерация миграций (`--autogenerate`)
- Ручное создание миграций
- Структура файлов миграций (upgrade/downgrade)
- Ограничения автогенерации

### Пример 4: Управление версиями и ревизии

**Цель:** Продемонстрировать работу с версиями и зависимостями миграций.

**Что включено:**
- Модели `Category` и `Book` со связями
- Примеры переходов между версиями
- Демонстрация параметров revision, down_revision, depends_on

**База данных:** `demo_revisions_db`

**Ключевые концепции:**
- Управление версиями миграций
- Переход к конкретной ревизии
- Относительные переходы (+N, -N)
- Зависимости между миграциями
- Просмотр истории миграций

### Пример 5: База данных - наполнение данными (seeding)

**Цель:** Показать, как заполнять БД начальными данными через миграции.

**Что включено:**
- Модели `Country` и `City`
- 3 миграции: создание таблиц, заполнение стран, заполнение городов
- Примеры использования `op.bulk_insert()`

**База данных:** `demo_seeding_db`

**Ключевые концепции:**
- Seeding через миграции
- Метод `op.bulk_insert()`
- Метод `op.execute()` для SQL
- Откат данных в downgrade()
- Порядок заполнения связанных таблиц

### Пример 6: Откаты миграций

**Цель:** Научить безопасно откатывать миграции.

**Что включено:**
- Модель `Employee` с постепенным добавлением полей
- 5 последовательных миграций
- Примеры различных сценариев откатов

**База данных:** `demo_rollbacks_db`

**Ключевые концепции:**
- Откат на N версий назад
- Откат к конкретной ревизии
- Полный откат (downgrade base)
- Правила написания downgrade()
- Безопасность откатов
- Типичные ошибки при откатах

## 📋 Полезные команды Alembic

```bash
# Инициализация Alembic
alembic init alembic

# Создание миграции (автоматически)
alembic revision --autogenerate -m "описание"

# Создание пустой миграции (вручную)
alembic revision -m "описание"

# Применить все миграции
alembic upgrade head

# Применить N миграций вперёд
alembic upgrade +N

# Откатить одну миграцию
alembic downgrade -1

# Откатить N миграций
alembic downgrade -N

# Откатить к конкретной ревизии
alembic downgrade <revision_hash>

# Откатить все миграции
alembic downgrade base

# Просмотр истории миграций
alembic history

# Подробная история
alembic history --verbose

# Текущая версия БД
alembic current

# Детали конкретной миграции
alembic show <revision_hash>
```

### Типичные проблемы студентов

**Проблема:** Alembic не видит изменения в моделях  
**Решение:** Проверить импорт моделей в env.py, убедиться что `target_metadata = Base.metadata`

**Проблема:** Ошибка подключения к БД  
**Решение:** Проверить DATABASE_URL, убедиться что PostgreSQL запущен

**Проблема:** Миграция не откатывается  
**Решение:** Проверить функцию downgrade(), убедиться что она не пустая

**Проблема:** Автогенерация создаёт пустую миграцию  
**Решение:** Убедиться что модели импортированы в env.py
